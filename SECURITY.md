# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately by email:

contact@xelis.io

For sensitive reports, encrypt your message with the PGP public key below.

Do not open a public issue for suspected vulnerabilities until the issue has
been reviewed and coordinated.

## What to Include

Please include as much of the following as possible:

- Affected component or binary.
- A clear description of the issue and impact.
- Steps to reproduce, proof of concept, or exploit conditions.
- Relevant logs, transaction/block data, configuration, or network details.
- Whether the issue may affect funds, consensus, privacy, RPC/API access, or
  peer-to-peer networking.
- Any suggested mitigation, if known.

## Scope

Reports involving the following areas are especially important:

- Consensus validation and block acceptance.
- Transaction verification, replay protection, and double-spend prevention.
- Cryptographic implementations and zero-knowledge proof verification.
- Peer-to-peer networking and denial-of-service risks.
- Build, release, and dependency supply-chain security.

## Coordination

The project will review reports privately and coordinate disclosure timing based
on severity, exploitability, and remediation status. Please avoid sharing
technical details publicly until a fix or mitigation is available.
